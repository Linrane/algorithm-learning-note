# Chapter1 初识算法

## 1.1 算法无处不在

算法其实就是**一种方法**，例子有：查字典，因为查字典这个动作本身就是一种方法（二分查找），比如整理扑克牌，从无序到有序的过程（插入排序），货币找零（贪心算法），算法，无处不在！

## 1.2 算法是什么

在刚刚我给出了一个很简短的概括，算法是一种方法，是什么样的方法，**在有限时间内解决特定问题的一组操作指令和步骤**，这就是算法，问题是什么样的问题，必须是**明确的具体的**，不能是含糊不清的，无法落地的，有限限制了**时间步骤和内存空间**，每个步骤都是**确定的**，在相同的状态下运行，得到的结果也必须是**确定的**。

数据结构就是用来**管理和存储信息**的，自然就是怎么样**高效**怎么来，**占用少**，**步骤少**，**操作便捷**。

数据结构提供了**结构化的数据**，而算法则是可以读取数据来达到想要的目的，而要实现目的，算法和数据结构都需要进行不同的调整。

# Chapter2 复杂度分析

## 2.1 算法效率评估

两个目的，一个用来**找到答案**，一个用来找到答案的最优解，这就是我们需要用算法需要做的。

那么，如何评估效率？两个维度，第一个是时间的长短，第二个是空间（占用内存空间的大小）

既然有了目的，有了效率评估，我们具体应该怎么做，有两个，实际测试和理论估算

实际测试虽然能很好的反映情况，但是有两个缺陷，第一个是没法排除环境干扰因素，你不可能在各种设备上都测试一遍，并且不同的算法会有不同的优势并且不同的计算机的硬件也会有不同程度的影响；第二点测试是十分耗费资源的，要完整展开测试是需要太久太久的时间了，不可能一时间得到一个合适的答案，哪怕得到了答案，也不一定有所谓说服力。

理论估算，这种方法被称之为渐进复杂度分析，简称复杂度分析，复杂度分析能够体现算法运行所需的时间和空间资源与输入数据规模之间的关系。它描述了随着输入数据规模的增加，算法执行所需时间和空间的增长趋势。也就是说，克服了很多弊端！

## 2.2 迭代与递归

迭代就是在一定条件下的重复，在编程中，就是在满足条件下的程序不断执行程序步骤的过程，其中若不满足条件，则会停止运行。

（本质上就是初中和高中数学题上的程序设计类的问题，只不过题目中的是可视化的表达，而现在是实打实的程序）

递归就是不断调用自身这个函数，继续小下去，直到终止，并且到达终止以后，从最深层的函数开始逐层返回。也就是自下而上地解决问题。

调用栈，其实就是函数每次在调用自身的时候就会开启一个新的函数来存储内容，直到函数结束后才能被清楚（都储存在栈帧空间当中），所以递归的内存会更多，时间效率更慢。

如果函数在返回前的最后一步才进行递归调用，可以被编译器和解释器优化，就被称之为尾递归，在效率上与迭代相当。
